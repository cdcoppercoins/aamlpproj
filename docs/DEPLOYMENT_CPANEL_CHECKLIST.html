Pre-deployment checklist for WHM / cPanel — read before changing anything on the server

Purpose: Work through this checklist in the break room before uploading files or changing live settings. This app is Laravel 12 and requires PHP 8.2 or newer. Your legacy site can stay on an older PHP version on the same server as long as each domain uses the correct PHP version in cPanel.

Not in Git: .env, vendor/, public/plates/, and MySQL data must be handled separately.

Overview — what lives where

Item In Git? How it reaches production
Laravel code, views, CSS, banners Yes Git clone, ZIP upload, or deploy script
.env (secrets, DB, mail) No Create manually on server
vendor/ (Composer packages) No composer install on server (needs SSH usually)
public/plates/ (plate photos) No ZIP, File Manager, or rsync
MySQL data (plates table, etc.) No phpMyAdmin import or mysqldump

What happens after this checklist

  1. Create subdomain and point document root at .../public
  2. Create production database and .env
  3. Upload code and run composer install --no-dev
  4. Import local MySQL dump and upload public/plates/
  5. Set permissions on storage/ and bootstrap/cache/
  6. Test on subdomain, then switch main domain when ready
  7. Set up one-click deploy (Git push + script or GitHub Action)
