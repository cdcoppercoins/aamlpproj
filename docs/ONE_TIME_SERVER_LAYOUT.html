One-time server layout

Do this once. After this, you never use new.minilicenseplates.com again.

Goal: Visitors use public_html only. PHP code lives in a sibling folder named laravel (not on the web). Future deploys upload to those two folders — no third path.

How it will look on the server

/home/minilp/
  laravel/           ← app, vendor, .env (NOT public on the web)
    app/
    vendor/
    storage/
    ...
  public_html/       ← minilicenseplates.com loads this folder
    index.php        ← points to ../laravel
    main.css
    plates/
    hero/
    ...
Before you start: The site works now. Download a backup of public_html and new.minilicenseplates.com to your PC with FileZilla (optional but recommended).

Steps (cPanel File Manager)

  1. Step 1 — Rename the Laravel folder
    In File Manager, go to /home/minilp.
    Right-click new.minilicenseplates.com → Rename → type: laravel → Save.
  2. Step 2 — Fix public_html/index.php
    Open public_html/index.php → Edit.
    Replace all contents with the file on your PC:
    d:\aamlpproj\deploy\public_html-index.php
    (It uses /laravel instead of /new.minilicenseplates.com.)
    Save.
  3. Step 3 — Copy any missing web files
    Open laravel/public/ on the server.
    If something is newer there than in public_html (e.g. main.css, js/, hero/), copy it into public_html (overwrite if newer).
    Do not copy the whole plates/ folder unless photos are missing from public_html/plates/.
  4. Step 4 — Test
    Open https://minilicenseplates.com → Ctrl+F5.
    Click Gallery and Search once.
  5. Step 5 — Done
    Folder new.minilicenseplates.com should be gone (renamed to laravel).
    From now on, use docs/DEPLOY_EASY.md for every update.

Every update after this

On your PC:

cd d:\aamlpproj
.\scripts\Build-DeployRelease.ps1 -OpenFolder

Upload:

  • laravel\ → server /laravel/
  • public_html\ → server /public_html/ (includes index.php)

Or upload release.zip to /home/minilp and extract once (both folders).

If the site breaks

Rename laravel back to new.minilicenseplates.com and put the old line back in index.php (or restore from backup).
