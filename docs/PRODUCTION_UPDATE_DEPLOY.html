Why two folders existed (and why we fix it)

Normal Laravel hosting works like this:

your-app-folder/          ← PHP code, .env (NOT visible on the web)
  app/
  resources/
  public/                   ← ONLY this folder is on the internet
    main.css
    index.php
    plates/

When the new site was built, Laravel was put in new.minilicenseplates.com/, but minilicenseplates.com still pointed at the old public_html/ folder. That forced you to upload CSS twice. Part A points the domain at Laravel’s public folder — the standard, correct setup.

The folder name new.minilicenseplates.com is just an old label on the server. It is not a website address. Your site stays minilicenseplates.com.

Part A   One-time fix (after Part 0 is done)

Allow about 30–45 minutes. Do not start Part A until Part 0 is finished — you want the backup to capture the updated site, not the old one.

  1. Back up both server folders to your PC Open FileZilla → connect. On the right (Remote) side at /, download:
    • public_html → e.g. d:\aamlpproj-backups\public_html-backup (usually quick)
    • new.minilicenseplates.com → e.g. d:\aamlpproj-backups\laravel-backup — skip public/plates/ (tens of thousands of photos; already on server and unchanged). In FileZilla you can download folder-by-folder and omit public/plates, or cancel if the queue only contains plate images.
    You are backing up a snapshot of the updated site before the cPanel change — not downloading plate photos again.
  2. Copy web files from public_html into Laravel’s public folder Still in FileZilla, open both folders side by side:
    • Remote: /public_html/
    • Remote: /new.minilicenseplates.com/public/
    Copy these from public_html into new.minilicenseplates.com/public/ only if the file is missing or older in the Laravel folder. When FileZilla asks, choose Overwrite if newer or Overwrite:
    • main.css
    • js/ folder (entire folder)
    • plates/ — skip unless gallery photos are missing from new.minilicenseplates.com/public/plates/
    • Banner images: home_top_banner.jpg, blue_back_composite_img_sm.jpg, header_banner.png, background_img.jpg, etc.
    • setinfo/ folder if present
    • hero/ folder if present
    • plate_missing.png if present
    Do not copy the old public_html/index.php — Laravel has its own index.php in public/. That is correct.
  3. Log in to cPanel Open your hosting login page (often https://minilicenseplates.com/cpanel or a link from your host’s welcome email). Sign in with your hosting username and password (not your FileZilla password unless they are the same).
  4. Change the document root for minilicenseplates.com This tells the internet: “serve minilicenseplates.com from Laravel’s public folder.”

    Where to click (cPanel varies slightly)

    • Domains (or Domains → Domains) → find minilicenseplates.com → click Manage or the pencil/edit icon
    • Look for Document Root or Root Domain Document Root
    • Change it from public_html to:
    new.minilicenseplates.com/public

    Some cPanel screens show the full path, e.g. /home/chuck/new.minilicenseplates.com/public — that is the same thing. Click Save or Update.

    Can’t find Document Root? Open a support ticket with your host and paste this exact request:

    Please change the document root for minilicenseplates.com
to: new.minilicenseplates.com/public
(Laravel application — standard setup)
  5. Set PHP version (while you are in cPanel) Go to MultiPHP Manager (or Select PHP Version). Set minilicenseplates.com to PHP 8.2 or 8.3. Save.
  6. Test the live site Open https://minilicenseplates.com in a new browser tab. Press Ctrl+F5 (hard refresh). Check:
    • Home page loads with styling (not plain unstyled HTML)
    • https://minilicenseplates.com/main.css opens and shows CSS text
    • Gallery opens and plate photos appear
    • Sign-in / register pages work
    If all good, Part A is done. You never need to upload to public_html again.
  7. If the site breaks — undo in 30 seconds Go back to cPanel → Domains → minilicenseplates.com → change Document Root back to public_html → Save. The site returns to the old setup. Then ask your host for help with the document root change.
  8. Optional: rename the old public_html folder (after a few days) Once you are confident the site works, in FileZilla rename public_html to public_html_OLD_DO_NOT_USE so you are not confused later. Do not delete until you are sure.

Part B   Every update after Part A (your normal routine)

After Part A, one rule: upload changed files to the same path under /new.minilicenseplates.com/ on the server.

On your PC (d:\aamlpproj\…)Upload to on server
app/, config/, resources/, routes/, database//new.minilicenseplates.com/ (same subfolders)
public/main.css, public/js/, images/new.minilicenseplates.com/public/
vendor/ (only after composer install on PC)/new.minilicenseplates.com/vendor/
public/plates//new.minilicenseplates.com/public/plates/ (only new photos)
.envNever upload
  1. Test locally first Start MySQL in XAMPP. In PowerShell: cd d:\aamlpproj then php artisan serve. Open http://localhost:8000 and confirm your changes work.
  2. Commit and push in GitHub Desktop Open GitHub Desktop → review changed files → write a short commit message → Commit to main → Push origin. This backs up your work on GitHub. The live site is updated in the next step, not by push alone.
  3. Upload with FileZilla Connect FileZilla. Left pane: d:\aamlpproj. Right pane: /new.minilicenseplates.com/. Drag only the files that changed to the same folder path on the server. When asked, choose Overwrite.

    Example: local public/main.css → remote /new.minilicenseplates.com/public/main.css

  4. New database changes only (migrations) If there are new files in database/migrations/ on your PC, upload them to /new.minilicenseplates.com/database/migrations/, then ask your host to run this once (or use a one-time cPanel Cron Job): 
    php /home/chuck/new.minilicenseplates.com/artisan migrate --force
    (Your host can confirm the exact path — it may differ slightly.)
  5. New PHP packages only (composer.json changed) On your PC: composer install --no-dev from d:\aamlpproj. Upload the whole vendor/ folder to /new.minilicenseplates.com/vendor/. Skip this if you only changed CSS or views.
  6. Test live Open https://minilicenseplates.com → Ctrl+F5. Check the pages you changed.

If pages look wrong after an upload

In FileZilla, under /new.minilicenseplates.com/:

  • Delete .php files inside bootstrap/cache/ (keep .gitignore)
  • Delete files inside storage/framework/views/

Hard refresh the browser again. No command line needed.

Never upload these from your PC

  • .env — the server has its own (with production database password and APP_URL=https://minilicenseplates.com)
  • mlp_code/ — old source code, not used
  • Entire public/plates/ unless you added new photos locally (already on server)
